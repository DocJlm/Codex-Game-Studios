# Claude to Codex Migration Guide

This guide explains how the upstream Claude Code Game Studios layout maps to Codex Game Studios.

## Runtime Mapping

| Upstream concept | Codex adaptation |
| --- | --- |
| `.claude/skills/<name>/SKILL.md` | `plugins/codex-game-studios/skills/cgs-<name>/SKILL.md` |
| Slash command `/name` | Skill invocation `$cgs-name` |
| `.claude/agents/*.md` | Role cards in `plugins/codex-game-studios/references/role-cards/` |
| `.claude/docs/templates/` | Templates in `plugins/codex-game-studios/assets/templates/` |
| `.claude/rules/` | Path rules in `plugins/codex-game-studios/references/rules/` |
| `.claude/settings.json` hooks | Not installed; preserved as reference scripts only |
| `CLAUDE.md` | `AGENTS.md` |

## Skill Adaptation Rules

- Prefix every migrated skill with `cgs-`.
- Keep only `name` and `description` frontmatter.
- Put original command aliases in the description, for example `/start` and `$cgs-start`.
- Replace legacy command references with `$cgs-*`.
- Replace legacy project context paths with plugin reference paths.
- Keep long templates and references outside `SKILL.md` so Codex loads them only when needed.

## Role Card Policy

Codex role cards are reference prompts, not guaranteed runtime subagents. A workflow may read a role card and apply that perspective during a sequential review. Parallel subagents are only appropriate when the user explicitly asks for parallel agent work and the current Codex session supports it.

## Regenerating From Upstream

Keep an upstream checkout beside this repository:

```text
D:\Git\Claude-Code-Game-Studios
D:\Git\Codex-Game-Studios
```

Then run:

```powershell
python tools\migrate_from_claude.py
python tools\validate_cgs.py
```

Manual docs such as `README.md`, `AGENTS.md`, and this guide are not regenerated by the migration script.
