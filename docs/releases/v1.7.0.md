# Codex Game Studios v1.7.0

v1.7.0 strengthens the example and smoke-fixture evidence loop so users can reproduce the path from `$cgs-start` through `$cgs-dev-story`, `$cgs-story-done`, and `$cgs-gate-check`.

## Changes

- Added `docs/transcripts/empty-game-smoke-run.md` as a no-write smoke transcript for `tests/fixtures/empty-game/`.
- Added `tests/fixtures/empty-game/production/gates/production-readiness.md` with an expected `BLOCKED` gate verdict.
- Updated empty-game README, walkthrough, and smoke checklist to include `$cgs-gate-check`.
- Updated Spark Sprint README, walkthrough, example doc, and transcript to include `$cgs-gate-check`.
- Extended `tools/validate_smoke_fixture.py`, `tools/validate_transcripts.py`, and `tools/validate_examples.py` to require the new gate-check evidence.
- Bumped plugin metadata to `1.7.0`.

## Validation

Windows:

```powershell
python tools\run_all_validators.py
```

macOS:

```bash
python3 tools/run_all_validators.py
```

Focused checks:

```bash
python3 tools/validate_smoke_fixture.py
python3 tools/validate_transcripts.py
python3 tools/validate_examples.py
```

## Compatibility Notes

- Skill, role-card, rule, template, plugin manifest, and marketplace paths remain unchanged.
- Runtime hooks remain disabled by policy.
- Godot remains optional for repository validation and CI.
- The empty-game fixture is intentionally blocked at the gate because it has planning evidence but no implementation evidence.
