# Codex Game Studios v1.5.0

v1.5.0 cleans up the testing framework path model so `$cgs-skill-test` and `$cgs-skill-improve` point at real repo-local files instead of the upstream root folder name.

## Highlights

- Replaced stale testing framework root references with `plugins/codex-game-studios/references/testing-framework/...`.
- Renamed the testing framework instruction file to `plugins/codex-game-studios/references/testing-framework/AGENTS.md`.
- Updated the testing framework catalog so every `spec:` path resolves from the repository root.
- Fixed `$cgs-skill-test` and `$cgs-skill-improve` instructions to use `plugins/codex-game-studios/skills/cgs-[name]/SKILL.md`.
- Added `tools/validate_testing_framework_paths.py` and wired it into `tools/run_all_validators.py`.
- Bumped plugin metadata to `1.5.0`.

## Validation

Windows:

```powershell
python tools\run_all_validators.py
python tools\validate_testing_framework_paths.py
```

macOS:

```bash
python3 tools/run_all_validators.py
python3 tools/validate_testing_framework_paths.py
```

Godot-specific runtime coverage remains in `tools/validate_godot_example.py`.

## Compatibility

- Existing `cgs-*` skill names remain stable.
- The plugin manifest path and marketplace path remain unchanged.
- Runtime hooks remain intentionally disabled.
- Testing framework specs remain optional reference material.
