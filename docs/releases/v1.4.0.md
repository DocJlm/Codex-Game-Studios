# Codex Game Studios v1.4.0

v1.4.0 converts the remaining full-migration skills into Codex-native workflows. The skills now describe role-card reviews, explicit validators, and current-session execution instead of relying on Claude subagent or Task-tool runtime assumptions.

## Highlights

- Replaced the old full-migration banner in 50 skills with a shared `Codex Operating Notes` section.
- Clarified that role perspectives come from `plugins/codex-game-studios/references/role-cards/`.
- Replaced direct `subagent_type`, `Task tool`, `$ARGUMENTS`, and subagent wording with Codex role-card review language.
- Added `tools/validate_codex_native_skills.py` and wired it into `tools/run_all_validators.py`.
- Bumped plugin metadata to `1.4.0`.

## Validation

Windows:

```powershell
python tools\run_all_validators.py
python tools\validate_codex_native_skills.py
```

macOS:

```bash
python3 tools/run_all_validators.py
python3 tools/validate_codex_native_skills.py
```

Godot-specific runtime coverage remains in `tools/validate_godot_example.py`.

## Compatibility

- Existing `cgs-*` skill names remain stable.
- The plugin manifest path and marketplace path remain unchanged.
- Runtime hooks remain intentionally disabled.
- Role cards remain references, not automatic subagents.
