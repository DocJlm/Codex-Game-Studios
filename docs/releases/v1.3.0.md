# Codex Game Studios v1.3.0

v1.3.0 adds an upstream parity report and a validator that keeps the Codex-native port honest about what is complete, what is intentionally adapted, and what remains on the roadmap.

## Highlights

- Added `docs/upstream-parity.md` with the checked upstream commit and surface-count matrix.
- Documented why the local template count is 40: the checked upstream tree also contains 40 Markdown templates despite the upstream README advertising 41.
- Mapped all 12 upstream hook scripts to Codex-native replacements without installing runtime hooks.
- Added `tools/validate_upstream_parity.py` and wired it into `tools/run_all_validators.py`.
- Bumped plugin metadata to `1.3.0`.

## Validation

Windows:

```powershell
python tools\run_all_validators.py
```

macOS:

```bash
python3 tools/run_all_validators.py
```

The validator checks local counts, the parity report tokens, the preserved hook reference scripts, and the no-runtime-hooks plugin policy.
Godot-specific runtime coverage remains in `tools/validate_godot_example.py`.

## Compatibility

- Existing `cgs-*` skill names remain stable.
- The plugin manifest path and marketplace path remain unchanged.
- Runtime hooks remain intentionally disabled.
- Godot remains optional for repository validation and CI.
