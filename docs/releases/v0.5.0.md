# Codex Game Studios v0.5.0

v0.5.0 evaluates Codex runtime hook support and turns the project policy into a CI-backed check.

## Highlights

- Added `docs/hooks/runtime-hook-evaluation.md` with the current hook decision, rationale, legacy hook mapping, and future adoption gate.
- Confirmed that Codex Game Studios does not bundle plugin runtime hooks in this release.
- Added `tools/validate_hook_policy.py` to enforce no `hooks` field in the plugin manifest and no plugin-root `hooks/` directory.
- Wired hook policy validation into GitHub Actions.
- Bumped regenerated plugin metadata to `0.5.0`.

## Decision

Codex plugin hooks exist, but plugin-bundled hooks are off by default unless users enable `[features].plugin_hooks = true`. For this project template, safety checks should behave the same in a fresh clone and in CI. Therefore v0.5 keeps safety behavior explicit through scripts and skill instructions.
