# Codex Game Studios v0.3.0

v0.3.0 turns the validated v0.2 package into a clearer demo surface. The release focuses on the first real usage loop: starting from a small concept, shaping the project, picking up a story, and closing it with review and QA follow-up.

## Highlights

- Added `docs/transcripts/concept-to-story.md`, a realistic transcript for Spark Sprint that walks through `$cgs-start`, `$cgs-setup-engine`, `$cgs-map-systems`, `$cgs-create-architecture`, `$cgs-create-epics`, `$cgs-create-stories`, `$cgs-dev-story`, and `$cgs-story-done`.
- Added `docs/workflows/high-frequency-workflows.md` to document the expected behavior of `$cgs-code-review`, `$cgs-qa-plan`, and `$cgs-smoke-check`.
- Tightened `$cgs-code-review` with severity rules and a findings-first output contract.
- Tightened `$cgs-qa-plan` with evidence rules and explicit test-matrix expectations.
- Tightened `$cgs-smoke-check` with a narrower smoke scope and clearer failure reporting.
- Added `tools/validate_transcripts.py` and wired it into CI.
